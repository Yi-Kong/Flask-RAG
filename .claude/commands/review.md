# /review — 代码审查命令

对修改的代码进行全面的安全和质量审查。

## 审查维度

### 1. 🔴 安全问题（最高优先级）

检查代码中是否存在：

- **硬编码密钥**：password、secret、token、api_key、api_token 等是否从环境变量读取
- **SQL 注入**：是否使用了字符串拼接构建 SQL（应使用 ORM）
- **XSS 漏洞**：是否对用户输入使用了 `|safe` 过滤器（Jinja2）
- **认证绕过**：新路由是否正确设置了认证中间件
- **文件上传风险**：是否校验了文件类型、大小、文件名
- **敏感信息泄露**：日志中是否打印了完整 API key、token、密码
- **权限校验缺失**：API 是否校验了数据归属（用户只能操作自己的数据）

### 2. 🟡 代码质量问题

- **重复代码**：是否存在可以提取为公共函数的重复逻辑
- **异常处理**：是否使用了 `exceptions.py` 中的自定义异常类
- **响应格式**：API 是否使用了 `ok()`/`fail()` 统一响应格式
- **输入校验**：Controller 层是否校验了所有用户输入
- **硬编码值**：URL、文件路径、配置值是否应从 config 读取
- **资源清理**：数据库连接、文件句柄是否正确关闭
- **事务管理**：数据库操作失败时是否正确回滚

### 3. 🟢 Flask 约定检查

- **Blueprint 注册**：新 Blueprint 是否在 `routers/router.py` 中注册
- **URL 生成**：模板中是否使用 `url_for()` 而非硬编码路径
- **请求方法**：GET/POST 使用是否正确（GET 不修改数据）
- **Cookie 安全**：是否设置了 `httpOnly`、`SameSite`、`secure`（生产环境）
- **Session 使用**：是否正确使用 `flask.g` 和 `flask.session`
- **错误处理**：是否依赖全局错误处理器而非 try-except 捕获所有异常

### 4. 🔵 模板问题

- **变量传递**：Controller 传给模板的变量是否在模板中正确使用
- **未定义变量**：模板中引用的变量是否在 Controller 中有传递
- **表单 CSRF**：表单是否需要 CSRF 保护（当前项目未使用 Flask-WTF，需关注）
- **JS 内联质量**：内联的 JS 是否可能导致 XSS 或 DOM 注入
- **Flash 消息**：错误/成功消息是否使用了 flash 机制

### 5. 🟣 数据库风险

- **N+1 查询**：是否在循环中查询数据库
- **缺少索引**：新表的查询条件字段是否有索引
- **外键约束**：外键关系是否正确配置，级联删除/更新是否合理
- **Migration 检查**：模型变更是否附带了迁移文件
- **数据兼容性**：新字段是否对旧数据兼容（nullable、default）

## 审查时的关键检查点清单

审查任何代码变更时，快速过一遍：

- [ ] `.env` 没有被提交
- [ ] 没有硬编码的密钥/密码/token
- [ ] API 响应使用了 `ok()` / `fail()`
- [ ] 异常使用了 `exceptions.py` 中的类型
- [ ] 新路由有正确的认证模式
- [ ] 文件上传有扩展名和大小校验
- [ ] SQL 查询使用了 ORM（无字符串拼接）
- [ ] 模板中 URL 使用了 `url_for()`
- [ ] Model 变更附带了 migration
- [ ] 没有引入前后端分离框架
- [ ] 没有引入不必要的大型新依赖
